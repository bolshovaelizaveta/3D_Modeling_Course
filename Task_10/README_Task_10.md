# Результаты работы
Поиск и регистрация заданных геометрических объектов на комплексной трехмерной сцене.
Объекты поиска: Снеговик, Стол, Пирамида, Шестилучевая звезда (из прямоугольников, конусов).
Метод: Global RANSAC (FPFH) + Local ICP (Point-to-Plane).
Качество регистрации: Fitness: > 0.99 (для всех объектов).
Inlier RMSE: ~0.015 (соответствует уровню синтетического шума сцены).
Все объекты успешно локализованы на сцене. Результаты визуализации и метрики сохранены в директории results/.

**Описание процесса и анализ результатов**

1. Использованы модели из Домашней работы работы №1 (Task 2). Объекты были трансформированы (перенос, вращение), объединены в единую сцену с добавлением плоскости и зашумлены Гауссовым шумом.
2. Препроцессинг. Выполнена вокселизация (Voxel Grid Downsampling) с шагом 0.1 и расчет гистограмм FPFH (Fast Point Feature Histograms) для описания локальной геометрии.
3. Глобальный поиск (Coarse Registration). Реализован алгоритм RANSAC. Для повышения надежности на плоских объектах (звезды) количество итераций увеличено, а критерий проверки (Edge Length check) установлен на 0.9. 
4. Визуализация в Plotly дает корректное пространственное совмещение шаблонов с объектами на сцене. Matplotlib не давал адекватного результата. 
   
**Целью данной работы является** реализация алгоритмов регистрации облаков точек для решения задачи "Object Picking" (поиск известного объекта в неструктурированной среде).

**Технические особенности реализации**

1. Адаптация под ARM64. Всё оптимизировано для работы на cpu. Стандартные методы визуализации Open3D (draw_geometries) заменены на интерактивную визуализацию через библиотеку Plotly. Сохранены результаты в HTML-формате. Удалось избежать ошибок графического контекста (Segmentation Fault) на macOS.
2. Масштабирование и центрирование. Для корректной работы дескрипторов FPFH критически важно совпадение масштабов. Была реализована логика предварительного центрирования шаблонов (manual_center) перед подачей в алгоритм регистрации.
3. Обработка плоских объектов. Стандартный размер вокселя (0.3), подходящий для крупных объектов (стол, снеговик), оказался велик для тонких лучей звезд. Параметр voxel_size был уменьшен до 0.1, чтобы корректно захватывать геометрию мелких деталей.
